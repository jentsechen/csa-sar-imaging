# sar-ship-detect — HRSID

SAR ship-detection dataset (HRSID) prepared for YOLO/ultralytics training and
evaluation. **The image data is not committed to git** (see `.gitignore`) — this
README + `convert_hrsid_to_yolo.py` make it fully reproducible from the source.

## Dataset: HRSID (High-Resolution SAR Images Dataset)
- **5,604 images**, 800×800 px, **16,969 ship instances**, single class `ship`
- Sources: **Sentinel-1 + TerraSAR-X**; resolutions 0.5 m / 1 m / 3 m
- Splits: 3,642 train (11,047 ships) · 1,962 test→val (5,922 ships)
- Annotations: **COCO JSON** (bounding boxes + segmentation masks)

> Note: HRSID is Sentinel-1/TerraSAR-X. A detector trained on a different sensor
> (e.g. Umbra/Capella) will show a domain-shift drop when evaluated here.

## Results
A YOLO11m detector trained on this dataset reaches **mAP@0.5 = 0.922** /
**mAP@0.5:0.95 = 0.702** on the HRSID test split. Full config, metrics, training
curves, and the reproduce command: **[docs/TRAINING.md](docs/TRAINING.md)**.

### Citation
S. Wei et al., "HRSID: A High-Resolution SAR Images Dataset for Ship Detection
and Instance Segmentation," *IEEE Access*, 2020.

## Source / how to re-download
Official repo: https://github.com/chaozhong2010/HRSID (download links live in its README).

This copy is the **JPG version** from Google Drive (file id `1BZTU8Gyg20wqHXtBPFzRazn_lEdvhsbE`):
```bash
pip install gdown
gdown 1BZTU8Gyg20wqHXtBPFzRazn_lEdvhsbE -O HRSID_JPG.zip
```
⚠️ Despite the `.zip` name, the file is actually a **RAR archive**. Extract with
`unrar`, `7z`, or `bsdtar`:
```bash
bsdtar -xf HRSID_JPG.zip      # -> HRSID_JPG/{JPEGImages,annotations}
```

## Layout
```
HRSID_JPG/                       raw download (gitignored)
  JPEGImages/                    5,604 .jpg
  annotations/                   train2017.json, test2017.json, train_test2017.json (COCO)
HRSID_YOLO/                      generated by the converter (gitignored)
  images/{train,val}/            symlinks into HRSID_JPG/JPEGImages (no duplication)
  labels/{train,val}/            YOLO .txt (class 0 = ship)
  data.yaml                      ultralytics dataset config
convert_hrsid_to_yolo.py         COCO -> YOLO converter (committed)
```

## COCO → YOLO conversion
```bash
python convert_hrsid_to_yolo.py
# train: 3642 images, 11047 ship boxes
# val  : 1962 images, 5922 ship boxes
```
HRSID's `test2017` split is mapped to YOLO `val`. Images are **symlinked**, not
copied, so the YOLO tree adds only kilobytes.

## Train / evaluate (ultralytics)
```bash
# train from scratch / fine-tune
yolo detect train model=yolov8m.pt data=HRSID_YOLO/data.yaml imgsz=640 epochs=100

# evaluate an existing model on the HRSID val split
yolo detect val model=path/to/best.pt data=HRSID_YOLO/data.yaml
```
